Comprehensive Mobile Test Framework

A complete test automation suite for a native Android e-commerce application, built from the ground up with Python, Appium, and Pytest.

Project Overview

This project is a robust, end-to-end mobile test automation framework designed to validate the quality of the Sauce Labs "My Demo App" for Android. The suite is built with a modern Python technology stack and follows industry best practices, including the Page Object Model (POM), to create a scalable and maintainable testing solution.

The framework goes beyond basic functional checks to include a comprehensive set of negative, edge-case, and non-functional tests. This holistic approach ensures not only that features work as expected, but also that the application is reliable, performant, and accessible. All test runs generate detailed, interactive HTML reports using the Allure Framework, providing clear visibility into test results and application quality.

Key Features & Testing Scope

  • Page Object Model (POM): The framework is architected with a clean separation of test logic and UI interaction, making it easy to update locators and maintain tests as the application evolves.
  • Functional Test Coverage: Includes end-to-end tests for critical user flows such as user login, product sorting (ascending and descending), adding multiple items to the cart, and the complete checkout process.
  • Negative & Edge-Case Testing: Validates application robustness by testing scenarios like invalid login credentials, submitting empty forms, and attempting to check out with an empty cart.
  • Data Validation: A dedicated test verifies the accuracy of the shopping cart's total price calculation by adding multiple items and asserting the sum.
  • Non-Functional Testing:
    • Performance: Measures application startup time to ensure it meets a defined performance threshold.
    • Accessibility: Verifies that key interactive elements have `content-description` labels for screen reader compatibility.
    • Reliability: Tests the app's ability to maintain state and user data after being sent to the background and reopened.
  • Advanced Reporting: Integrated with the Allure Framework to produce detailed, interactive HTML reports that categorize test results and provide clear insights into failures.
