End-to-End QA & DevOps Pipeline

A full-stack testing project demonstrating a complete CI/CD pipeline with UI, API, Contract, Performance, and Security testing layers.

Project Overview

This project showcases a complete, modern QA and DevOps pipeline built from the ground up. It intentionally separates the system-under-test into two components: a self-hosted, Dockerized **Express.js backend API** that I wrote, and an external, public **React frontend (TodoMVC)**. This separation allows for the demonstration of a full spectrum of SDET skills in isolation, proving capabilities across the entire testing pyramid.

The entire lifecycle is automated via a comprehensive **GitHub Actions workflow**. On every pull request, the workflow builds the backend API into a Docker container, runs health checks, and then executes multiple layers of automated tests against it. This ensures that every change is automatically validated for correctness, performance, and security, providing a robust quality gate before any code is merged.

Key Features & Pipeline Stages

  • UI Automation: Cross-browser UI tests written with Playwright and the Page Object Model (POM) run against the live TodoMVC React application, validating real-world user flows.
  • API Functional Testing: A dedicated API test suite uses Playwright's built-in `request` context to directly test the containerized Express.js backend, asserting endpoint correctness and schema validation.
  • Consumer-Driven Contract Testing: A Pact consumer test (using Jest) generates a contract, ensuring the API adheres to the JSON schema expected by its clients and preventing breaking changes.
  • Performance Guard-Rail: A k6 smoke test applies a 30-second load to the API. The CI job automatically fails the build if the p(95) latency exceeds a 500ms threshold, enforcing a strict performance budget.
  • Automated Security Scanning: An OWASP ZAP baseline scan is run against the live API within the CI pipeline. The build is blocked if any Medium or High-level vulnerabilities are discovered, embedding security directly into the development process.
  • DevOps & Containerization: The project includes a multi-stage Dockerfile for a lean production image, and the GitHub Actions workflow manages the entire build, deployment, and testing process within a self-contained environment.
