Full-Stack Selenium Test Framework

A comprehensive, containerized testing framework for the AutomationExercise e-commerce site, featuring a full CI/CD pipeline with UI, API, and Performance tests.

Project Overview

This project is a complete, end-to-end test automation solution built on a powerful Java and Selenium stack. It is designed to perform multi-layered testing against the `automationexercise.com` platform, validating everything from the UI and APIs to backend performance. The framework is architected for scalability, maintainability, and full integration into a modern DevOps lifecycle.

A key feature is its full CI/CD integration using GitHub Actions. The workflow, defined in `main.yml`, automates the entire testing process. It builds the project, runs the comprehensive test suite (including UI, API, and performance tests) inside a Docker container, and generates detailed reports. This ensures that every code change is automatically and thoroughly vetted, providing rapid feedback and maintaining a high standard of quality.

Key Features & Implementation

  • CI/CD and DevOps Pipeline: A GitHub Actions workflow automates the build and test cycle. The entire test environment is containerized using a `Dockerfile`, ensuring consistency and portability.
  • API Testing with RestAssured: The framework includes a dedicated API testing layer (`APITests.java`) that uses RestAssured to directly validate backend endpoints, ensuring data integrity and service reliability independent of the UI.
  • Performance Testing with JMeter: A JMeter test plan (`GetProducts.jmx`) is integrated into the project to conduct load testing on critical API endpoints, identifying performance bottlenecks under user load.
  • Data-Driven UI Testing: The robust UI test suite uses Selenium WebDriver and the Page Object Model (POM). It leverages TestNG's `@DataProvider` and Apache POI to read test cases from external Excel sheets, enabling extensive test coverage with minimal code duplication.
  • Advanced Reporting and Listeners: Custom TestNG listeners (`ExtentReporterNG`, `ScreenshotListener`) are used to generate detailed ExtentReports. These reports include step-by-step logs and automatically attach screenshots for any failed test, facilitating rapid debugging.
  • Thread-Safe Parallel Execution: A custom `DriverFactory` uses `ThreadLocal` to manage WebDriver instances, allowing for safe and efficient parallel execution of UI tests, significantly reducing the overall runtime of the test suite.
